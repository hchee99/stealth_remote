# 🕶️ 스텔스 플레이어 (Stealth Player)

업무 중 모니터 위에 영상/방송을 띄워두고, 필요할 때 즉시 화면에서 숨길 수 있게 해주는
Windows용 PyQt5 데스크톱 오버레이 프로그램입니다. 흔히 말하는 **"보스키(Boss Key)"** 형태의
스텔스 미디어 플레이어로, 평소엔 화면 위에 투명하게 깔려 있다가 단축키 한 번으로 통째로 사라지는
것이 핵심 기능입니다.

---

## 1. 만든 맥락

사무실/회사 PC에서 모니터 한쪽에 유튜브 영상이나 라이브 방송(치지직 등)을 켜두고 싶지만,
- 누가 지나가거나 화면을 보면 바로 들키고
- 매번 창을 최소화/복구하는 것도 번거롭고
- 알트탭으로 끄면 소리는 계속 나오는 문제가 있어서

→ **"평소엔 거의 안 보이게 투명하게 깔아두고, 위험하면 키 하나로 전부 즉시 숨긴다"**는
컨셉으로 만들어진 개인용 유틸리티입니다. 내장 브라우저 모드와, 이미 떠 있는 외부 창(크롬 PIP 등)을
통째로 투명/항상위로 만들어버리는 모드 두 가지를 지원합니다.

> ⚠️ 참고: 이 도구는 화면을 숨기는 기능을 제공할 뿐, 회사 보안 정책이나 사내 규정 위반 여부와는
> 별개의 문제입니다. 사용 환경의 사내 정책을 먼저 확인하고 사용하는 것을 권장합니다.

---

## 2. 핵심 동작 원리

| 요소 | 설명 |
|---|---|
| 항상 위 + 투명도 | `Qt.WindowStaysOnTopHint` + `setWindowOpacity()`로 창을 다른 작업창보다 위에, 그러나 흐릿하게 표시 |
| 외부 창 스텔스화 | Windows API(`ctypes.windll.user32`)로 **임의의 다른 프로그램 창**도 레이어드 윈도우로 바꿔 투명도·항상위 적용 |
| 전역 단축키 | `keyboard` 라이브러리로 OS 전역 핫키 등록 (창이 포커스 없어도 동작) |
| 설정 영속화 | 모든 옵션을 `stealth_merged_config.json`에 저장해 재실행 시 그대로 복원 |

---

## 3. 주요 기능

### 3-1. 내장 브라우저 모드 (📺)
- `QWebEngineView`로 유튜브 등의 URL을 직접 로드하는 내장 브라우저 창
- 영상 로드가 끝나면 자바스크립트를 주입(`clean_ui`)해서
  - 유튜브의 헤더/탐색바/추천영상 등 UI 요소를 모두 숨기고
  - `<video>` 태그만 화면 전체를 채우도록 강제 (`MutationObserver` + 0.5초 간격 재적용으로 유튜브가 UI를 다시 그려도 계속 제거)
- 리모컨에서 투명도(0~100%) / 가로 크기(300~1000px, 16:9 비율 유지)를 실시간 슬라이더로 조절

### 3-2. PIP(외부 창) 모드 (🌐)
- "🌐 크롬 띄우기": 지정한 URL(기본값: 치지직 라이브)을 `--app` 모드의 별도 작은 크롬 창으로 실행
- "🎯 PIP창 타겟 (3초)": 버튼을 누르고 3초 안에 **숨기고 싶은 다른 창을 클릭**하면
  - `GetForegroundWindow()`로 그 창의 핸들(HWND)을 가져와
  - `WS_EX_LAYERED` 스타일 + `SetLayeredWindowAttributes`로 투명도를 줄 수 있는 창으로 변환
  - `HWND_TOPMOST`로 항상 위 고정
  - 이후 슬라이더로 그 창의 투명도를 따로 조절 가능
  - 즉, 크롬뿐 아니라 **PC에 떠 있는 거의 모든 창**을 같은 방식으로 투명화할 수 있음

### 3-3. 리모컨 위젯
- 메인 플레이어 아래에 따로 떠 있는 작은 컨트롤 패널 (드래그로 위치 이동 가능)
- 탭으로 "내장 브라우저 / PIP 모드" 전환, URL 입력창, 투명도·크기 슬라이더, 설정(⚙️) 버튼 포함

### 3-4. 전역 단축키 3종
| 키 (기본값) | 기능 |
|---|---|
| **F6** (panic_key) | **전체 패닉**: 메인 플레이어 + 외부 타겟 창 + 리모컨까지 전부 즉시 투명도 0 / 화면 밖으로 이동시키고 소리 음소거. 다시 누르면 패닉 이전 상태(투명도, 표시 여부)로 정확히 복원 |
| **INSERT** (hide_key) | **리모컨만 숨기기/보이기** 토글 (패닉 모드 중에는 무시됨) |
| **Alt+Q** (exit_key) | **완전 종료**: 모든 단축키 해제, 타겟 잡았던 외부 창을 원상복구(보이기 + 투명도 255 + 활성화 메시지 전송) 후 프로그램 종료 |

### 3-5. 설정 다이얼로그 (⚙️)
- 내장 브라우저 투명도, PIP창 투명도, 패닉/숨기기/종료 단축키를 GUI에서 직접 변경
- "저장 및 즉시 적용" 클릭 시 `stealth_merged_config.json`에 기록되고, 단축키도 즉시 재등록(`setup_global_shortcuts`)

---

## 4. 설정 파일 (`stealth_merged_config.json`)

```json
{
  "opacity": 55,        // 내장 브라우저 투명도 (0~100, %)
  "ext_opacity": 30,    // PIP/외부 타겟 창 투명도 (0~100, %)
  "panic_key": "F6",    // 전체 패닉 단축키
  "hide_key": "INSERT", // 리모컨 숨기기 단축키
  "exit_key": "Alt+q",  // 완전 종료 단축키
  "last_url": "...",    // 내장 브라우저에 마지막으로 로드했던 URL (자동 저장)
  "pip_url": "..."      // PIP 모드에서 마지막으로 사용한 URL (자동 저장)
}
```
프로그램이 종료될 때가 아니라 **URL을 입력하거나 설정을 저장하는 즉시** 파일에 기록되므로,
다음 실행 시 마지막 상태 그대로 이어집니다.

## 5. 실행 방법

### 5-1. 필요 환경
- **Windows** (외부 창 핸들 제어에 `ctypes.windll.user32` Win32 API를 직접 사용하므로 Windows 전용)
- Python 3.8 이상
- 크롬 브라우저 (PIP 모드의 "크롬 띄우기" 기능 사용 시)

### 5-2. 의존성 설치
```bash
pip install PyQt5 PyQtWebEngine keyboard
```

### 5-3. 실행
```bash
python claude_stealth.py
```
- 실행 폴더에 `stealth_merged_config.json`이 없으면 기본값으로 자동 생성됩니다.
- `keyboard` 라이브러리로 전역 단축키를 등록하기 때문에, 환경에 따라 **관리자 권한으로 실행**해야
  단축키가 정상 동작할 수 있습니다.

## 7. 알려진 제약 사항

- Windows 전용 (Win32 API 의존, macOS/Linux 미지원)
- `keyboard` 라이브러리의 전역 후킹 특성상 보안 소프트웨어가 키로거로 오인할 수 있음
- "🎯 PIP창 타겟"으로 잡은 외부 창이 자체적으로 항상-위 속성을 갖거나 일부 보호된 시스템 창일 경우
  레이어드 스타일 적용이 거부될 수 있음
- 패닉 모드 진입/해제 시 상태 복원은 "마지막 한 번의 상태"만 기억하므로, 복잡하게 모드를 전환한
  직후 패닉을 걸면 복원 결과가 의도와 다를 수 있음


# ☕ 축구 승률 커피내기 미니게임

팀원들끼리 "이번 경기 누가 이길까?"를 맞춰보고, 틀린 사람이 커피를 사는 간단한
**tkinter 기반 데스크톱 미니게임**입니다. 외부 라이브러리 설치 없이 Python 표준 라이브러리만으로
동작합니다.

---

## 1. 만든 맥락

축구 시즌(월드컵, 아시안컵 등)이나 사내 풋살 매치처럼 "누가 이길지" 화제가 되는 경기가 있을 때,
팀원들끼리 가볍게 **승부 예측 + 커피 내기**를 할 수 있도록 만든 미니게임입니다.
- 국가대표팀끼리의 경기뿐 아니라, `우리회사_개발팀` / `우리회사_영업팀`처럼
  **사내 팀 이름으로도 바로 바꿔 쓸 수 있게** 설계되어 있습니다.
- 누가 맞고 틀렸는지, 그래서 누가 커피를 사야 하는지를 자동으로 계산하고,
  과거 내기 기록도 파일로 남겨서 "지난번에 누가 제일 많이 샀는지" 확인할 수 있습니다.

---

## 2. 전체 동작 흐름

1. **팀 선택** → 두 팀의 사전 정의된 "전력 점수"로 승률(%)을 계산해 보여줌
2. **참가자 등록** → 함께 내기할 동료 이름을 추가
3. **베팅** → 참가자별로 둘 중 어느 팀이 이길지 예측 선택
4. **경기 시뮬레이션** → 승률에 맞춰 무작위로 승자를 결정
5. **결과 발표 + 기록 저장** → 맞춘 사람 / 커피 사야 하는 사람을 보여주고 JSON 파일에 누적 기록

코드 자체도 이 순서(① 데이터 → ② 계산 로직 → ③ 저장 → ④ 화면(GUI) → ⑤ 실행)대로
파일 안에 섹션이 나뉘어 있어서, 위에서부터 읽으면 흐름을 그대로 따라갈 수 있습니다.

---

## 3. 주요 기능 상세

### 3-1. 팀 전력 / 승률 계산 (`TEAM_POWER`, `calculate_win_rate`)
- 코드 상단의 `TEAM_POWER` 딕셔너리에 팀 이름과 전력 점수(숫자가 높을수록 강팀)가 미리 정의되어 있음
  - 예: 브라질 90, 독일 88, 대한민국 78, 사우디아라비아 60, 우리회사_개발팀 55, 우리회사_영업팀 50 등
- 승률 공식은 단순한 상대 점수 비율:
  ```
  A팀 승률 = A팀 전력 / (A팀 전력 + B팀 전력) × 100
  ```
- "승률 계산하기" 버튼을 누르면 두 팀의 승률(%)이 화면에 바로 표시됨

### 3-2. 경기 시뮬레이션 (`simulate_match`)
- 0~100 사이의 난수를 하나 뽑아서, 그 값이 A팀 승률보다 작으면 A팀 승리로 처리
- 승률이 높은 팀일수록 당첨 구간이 넓어서 더 자주 이기지만, 매번 결과가 랜덤이라
  약팀도 이길 수 있는 구조 (실제 스포츠의 변수를 흉내낸 단순 확률 모델)

### 3-3. 참가자 등록 & 베팅
- 이름을 입력하고 "추가" 버튼으로 내기 참가자 등록 (중복 등록 방지)
- "베팅 입력창 열기"를 누르면 등록된 참가자 수만큼 행이 있는 팝업창이 뜨고,
  각자 드롭다운으로 A팀/B팀 중 하나를 선택
- "베팅 확정하고 경기 시작!"을 누르면 곧바로 시뮬레이션 실행

### 3-4. 결과 발표
- 경기 시뮬레이션 결과(승리팀, 각 팀 승률)와 함께
  - 🎉 **맞춘 사람** 목록
  - ☕ **커피 사야 하는 사람**(예측이 틀린 사람) 목록
  을 팝업 메시지로 바로 보여줌

### 3-5. 기록 저장/조회 (`load_history`, `save_history`, `add_record`)
- 매 경기 결과는 `coffee_bet_history.json`에 다음 형태로 누적 저장됨:
  ```json
  {
    "날짜": "2026-06-23 15:06",
    "참가자_예측": { "성환": "대한민국", "정은": "일본" },
    "실제_승리팀": "일본",
    "커피_사는_사람들": ["성환"]
  }
  ```
- 프로그램을 껐다 켜도 과거 기록이 그대로 남아 있으며, 화면 하단 "④ 커피 내기 기록" 영역에
  **최근 10개** 기록을 최신순으로 보여줌

| ② 참가자 등록 | 이름 입력 + 추가 버튼 + 등록된 참가자 목록 |
| ③ 베팅 & 결과 확인 | 베팅 팝업창 열기 → 예측 선택 → 경기 시작 |
| ④ 내기 기록 | 최근 10개 내기 기록을 스크롤 가능한 텍스트 박스로 표시 |

## 5. 실행 방법

### 5-1. 필요 환경
- Python 3 (표준 라이브러리에 포함된 `tkinter`만 사용하므로 **추가 설치 불필요**)
  - 일부 Linux 배포판은 `tkinter`가 기본 미포함이라 별도 패키지 설치가 필요할 수 있음 (예: `sudo apt install python3-tk`)

### 5-2. 실행
```bash
python coffee_bet_game.py
```
실행 폴더에 `coffee_bet_history.json`이 없으면 빈 기록으로 자동 생성되고, 이후 내기마다 갱신됩니다.

## 6. 커스터마이징 가이드

- **팀/전력 점수 바꾸기**: 코드 상단 `TEAM_POWER` 딕셔너리만 수정하면 됨
  ```python
  TEAM_POWER = {
      "우리팀_프론트": 70,
      "우리팀_백엔드": 65,
      ...
  }
  ```
- **저장 파일 위치/이름 변경**: `SAVE_FILE` 변수 값만 바꾸면 됨
- **최근 기록 표시 개수 변경**: `refresh_log` 함수의 `self.history[-10:]` 부분의 숫자를 조절

## 8. 알려진 제약 사항

- 승률 계산이 단순 비율 기반이라, 실제 축구 통계(최근 폼, 부상자, 홈/어웨이 등)는 반영되지 않음
- 참가자 한 명이 베팅을 안 하거나 같은 사람이 여러 번 등록되는 경우에 대한 별도 예외 처리는 없음
- `coffee_bet_history.json`이 외부에서 손상된 형식으로 수정되면 `load_history()`에서 그대로
  예외가 발생할 수 있음 (별도 복구 로직 없음)
